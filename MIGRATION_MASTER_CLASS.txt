[CLASE_DE_INSTRUCCIONES_MIGRACION]
// Este archivo está diseñado para que la IA lo analice y ejecute la configuración del proyecto en un nuevo dispositivo.

OBJETIVO: Instalar y configurar el ecosistema "Aula Virtual Francisca Burgos" en una nueva laptop.

1. REQUISITOS_SISTEMA:
   - Node.js (v18 o superior)
   - Docker Desktop (Opcional, para base de datos local)
   - Git (Para clonar el repositorio)
   - VS Code

2. ESTRUCTURA_PROYECTO:
   - Root: Frontend (Vite + React)
   - /backend: Servidor Express (Node.js)
   - /supabase_schema.sql: Estructura de la base de datos.

3. PASOS_INSTALACION_FRONTEND:
   - Comando: `npm install` (en la carpeta raíz)
   - Archivo Config: Crear `.env.local` en la raíz con:
     VITE_GEMINI_API_KEY=[TU_LLAVE_AQUI]
     VITE_GROQ_API_KEY=[TU_LLAVE_AQUI]
     VITE_SUPABASE_URL=[TU_URL_SUPABASE]
     VITE_SUPABASE_ANON_KEY=[TU_KEY_ANONIMA]

4. PASOS_INSTALACION_BACKEND:
   - Comando: `cd backend && npm install`
   - Archivo Config: Crear `backend/.env` con:
     DATABASE_URL=postgresql://postgres:TU_CONTRASEÑA@db_host:5432/postgres
     PORT=4000

5. CONFIGURACION_BASE_DE_DATOS:
   - Opción A (Supabase): Ejecutar el contenido de `supabase_schema.sql` en el SQL Editor de Supabase.
   - Opción B (Docker): Ejecutar `docker-compose up -d` en la raíz para levantar Postgres y el Backend.

6. COMANDOS_EJECUCION:
   - Frontend: `npm run dev` (Desde la raíz)
   - Backend: `npm run dev` (Desde la carpeta /backend)

7. NOTAS_ADICIONALES:
   - El archivo `loader.js` se ejecuta automáticamente antes del frontend para mostrar el progreso.
   - Asegurarse de que el puerto 5173 (Vite) y 4000 (Backend) estén libres.

[FIN_DE_CLASE]
